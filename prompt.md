Prompt 1
========

Write me a Go program in src/release-tools/regression-chart that extracts and visualizes the following data stored by a git-bug bridge in .git/git-bug

1. Issues labeled "regression in VER" where VER is an version like 2.5.3 or 2.6.2.2
2. For these issues, when they are closed, the version milestone (also a VER)

This data shall be written to three JSON files in the following form:
1. For each found version, a list of regressions it introduced, and for the closed ones, the closing milestone.
2. For each milestone, a map from version to the number of regressions from this version the milestone closed.
3. For each milestone, a map from version to the number of regressions from this version that are still open at the time of this milestone.  (The milestone itself shall be included in the map as last entry.)

We then want to plot a diagram where on the x-axis we have the (version) milestones in order,
and for each milestone we want a bar indicating the number of regressions that were still open at this milestone.
The bar shall be segmented vertically into blocks so that the lowest block shall give the number of open regressions from the first version and the uppermost block the number of regressions from the last-known version at this time, i.e., this version.
Each version shall have its own color so that blocks with open regressions introduced in one version get the same color.

In the end, the chart should look similar to the one at https://camo.githubusercontent.com/985d56a3d4f69b8ff8857e4e0c2341f2b961037b4b149baa09d75db8a85d3b50/68747470733a2f2f636f6d6d65726369616c6861736b656c6c2e6769746875622e696f2f737461636b6167652d736e617073686f74732f63686172742e737667 generated by the Haskell program at https://github.com/commercialhaskell/stackage-snapshots/blob/master/stackage-history-chart/src/Main.hs (albeit the chart generated by this Haskell program plots stackage lts versions and not regressions).

The data underlying this diagram shall also be output as markdown table, looking e.g. like this (if there were only the versions 2.5.1.1, 2.5.2, 2.5.3):
```
Open regressions per version

|         | 2.5.1.1 | 2.5.2 | 2.5.3 | Total |
|---------|---------|-------|-------|-------|
| 2.5.1.1 |      12 |       |       |    12 |
| 2.5.2   |       7 |    10 |       |    17 |
| 2.5.3   |       5 |     2 |     4 |    11 |
```

Answer
------

● Implemented the tool in src/release-tools/regression-chart/main.go with a minimal go.mod. It reads regressions
  from local git-bug data, resolves closing milestones from GitHub, and writes:

   - regressions-by-version.json
   - regressions-closed-by-milestone.json
   - regressions-open-by-milestone.json
   - regressions-open-by-version.md
   - regressions-open-by-version.svg

  It defaults to -repo-root . -out-dir . and uses GITHUB_TOKEN, GH_TOKEN, or gh auth token for GitHub access.

  Two data quirks are handled explicitly: five issues carry multiple regression in ... labels, so they are
  counted under the earliest listed version and their full label set is preserved in JSON; one closed issue in
  this repo has no version milestone on GitHub, so it is surfaced under unclassified_closed_regressions and
  excluded from the milestone-based aggregates.
